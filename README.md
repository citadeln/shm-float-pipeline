# Высокопроизводительный канал сжатых данных

Данный проект реализует высокопроизводительный канал передачи сжатых данных через **разделяемую память POSIX** между процессами‑производителем (`Producer`) и процессом‑потребителем (`Consumer`). Исходные `float`‑данные считываются из файла, квантуются в `int16_t`, передаются по общему кольцевому буферу и затем восстанавливаются на стороне потребителя с измерением потерь.

## Описание приложения

- **Производитель (`producer`)**  
  Читает бинарный файл с `float`‑данными, квантует их в `int16_t` заданным масштабом, группирует в пакеты с заголовком `ChunkHeader` и передаёт через разделяемую память.
- **Потребитель (`consumer`)**  
  Получает пакеты из разделяемой памяти, декодирует `int16_t` обратно в `float` и записывает восстановленные данные в выходной бинарный файл.
- **Юнит‑тесты**  
  Проверяют корректность работы квантователя и кольцевого буфера.

## Требования

### Системные требования

- Linux или любая POSIX‑совместимая система с поддержкой `shm_open`, `sem_open`, `mmap`.
- Доступ к `/dev/shm` (стандартный `tmpfs`‑раздел для разделяемой памяти).

### Зависимости

- Компилятор C++20 (`g++` или `clang++`).
- `libpthread` (POSIX threads).
- `librt` (для `mmap`/`mlock` и семафоров, где требуется).
- CMake ≥ 3.16.
- Google Test (для `tests/test_utils`).

## Сборка проекта
```bash
./scripts/build.sh
```
Скрипт автоматически:
- создаёт каталог `build`,
- вызывает `cmake` с необходимыми флагами (C++20, флаги компилятора),
- запускает `make` с параллельной сборкой (`-j`).

## Запуск проекта
```bash
./scripts/run.sh
```
Скрипт:
- собирает проект (если ещё не собран);
- запускает `producer` и `consumer` параллельно;
- передаёт им входной файл `../data/sample.bin` и записывает результат в `../data/output.bin`;
- после завершения процессов запускает юнит‑тесты из `test_utils`.

Типичный вывод:

```text
Producer: 50ms, ratio: 0.500
Consumer: 45ms, loss: 0.00%
[==========] Running 3 tests from 2 test suites.
...
```

## Метод синхронизации процессов и используемые объекты
Для связи между процессами используется POSIX Shared Memory и именованные семафоры.

- Канал: Сегмент памяти `/test_channel` (256 байт), содержащий заголовок Header (указатели, емкость, магическое число) и кольцевой буфер данных.
- Семафоры:
    - `/test_empty`: сигнализирует о наличии свободных ячеек.
    - `/test_full`: сигнализирует о наличии новых данных.

## Порядок работы

1. **Инициализация**  
   Производитель вызывает `InitProducer`, создавая сегмент памяти (shm_open) и семафоры (sem_open).
   
2. **Передача данных**  
    - Производитель:
        - Ждет свободного места (операция sem_wait на /test_empty).
        - Записывает данные в ячейку буфера.
        - Сигнализирует о новых данных (операция sem_post на /test_full).
    - Потребитель:
        - Ждет появления данных (операция sem_wait на /test_full).
        - Читает данные из ячейки.
        - Освобождает место (операция sem_post на /test_empty).

4. **Завершение**  
   После обмена ресурсы закрываются, но объекты (shm_unlink, sem_unlink) не удаляются, чтобы не блокировать перезапуск системы. Очистка производится вручную или скриптами.

## Выбор алгоритма сжатия

В данном проекте **сжатие** реализовано как **квантование `float` в `int16_t`** с масштабным коэффициентом (`scale_`). 

Такой подход обеспечивает предсказуемую потерю точности при значительном сокращении объема данных за счет уменьшения разрядности

Выбор `int16_t` обусловлен компромиссом между размером и точностью.
`FloatQuantizer` обеспечивает отсечение значений за границами `int16_t` и линейное масштабирование.

## Компромиссы и оптимизации

- Буфер фиксированного размера: Упрощает синхронизацию, но требует, чтобы потребитель успевал обрабатывать данные, иначе производитель заблокируется.

- Потеря точности: Это осознанный компромисс ради уменьшения трафика. Погрешность (LossPercent) минимальна при правильном подборе масштаба.

- Очистка ресурсов: Объекты /test_channel, /test_empty, /test_full не удаляются в коде. Это предотвращает ошибки при перезапуске. Очистку нужно делать вручную перед запуском: `rm -f /dev/shm/test_*`
